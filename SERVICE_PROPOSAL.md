# 보라-부산(bora-busan) 서비스 개선 및 확장 제안서

본 문서는 현재 MVP 수준으로 구현된 보라-부산 서비스의 기획적 완성도를 높이고, 향후 스케일업을 대비하기 위한 개선 및 확장 제안을 담고 있습니다.

## 1. 블로그 및 콘텐츠 영역 고도화 (Blog UX/UI)

### 1.1. 실제 페이지네이션 또는 무한 스크롤 연동
- **현황**: 현재 `src/app/blog/page.tsx` 내에 `const totalPages = 1;`로 하드코딩되어 있습니다.
- **개선안**: 데이터베이스(Supabase)의 `count` API를 활용하여 전체 게시물 수를 가져오고, 이를 기반으로 실제 페이지네이션 로직을 구현합니다. 모바일 사용자 경험을 고려하여 무한 스크롤(Infinite Scroll) 방식 도입도 권장합니다.

### 1.2. 카테고리 기반 필터링 기능
- **현황**: 게시물마다 카테고리 데이터는 존재하나 필터링 UI가 제공되지 않습니다.
- **개선안**: 사용자가 관심 있는 카테고리(예: 맛집, 관광지, 성지 등)만 모아볼 수 있도록 블로그 목록 상단에 카테고리 필터(칩 형태 UI 등)를 추가해야 합니다.

## 2. 관리자 시스템(CMS) 완결성 확보

### 2.1. 게시글 수정(Update) 기능 구현
- **현황**: 관리자 대시보드에 게시글 목록과 삭제 기능은 구현되어 있으나, 수정 기능은 `Edit Post (Coming Soon)` 상태로 비활성화되어 있습니다.
- **개선안**: 작성된 글의 오타 수정이나 내용 업데이트를 위해 `[slug]` 또는 `[id]`를 기반으로 기존 데이터를 불러오고 수정할 수 있는 에디터 페이지 라우팅 및 연동 로직이 필요합니다.

### 2.2. 임시저장(Draft) 및 미리보기(Preview) 기능
- **현황**: 글 작성 후 즉시 발행(Publish)되는 구조로 추정됩니다.
- **개선안**: DB 테이블에 `status` 컬럼(예: 'draft', 'published')을 추가하여 작성 중인 글을 임시저장할 수 있는 기능을 도입합니다. 또한 발행 전 프론트엔드 환경에서 글이 어떻게 보이는지 확인할 수 있는 '미리보기' 기능을 제공하여 운영자의 실수를 방지해야 합니다.

## 3. 데이터 분석 및 마케팅 인프라 연동

### 3.1. 사용자 행동 분석 툴(GA4 등) 연동
- **현황**: 현재 유입 경로 및 사용자 행동 데이터를 추적하는 트래커가 부재합니다.
- **개선안**: 메인 페이지의 '뉴스레터 구독(리드 수집)' 및 '블로그 콘텐츠 조회'가 핵심 KPI이므로, Google Analytics 4 (GA4) 또는 Amplitude를 초기 단계부터 세팅하여 유저의 사이트 체류 시간, 이탈률, 구독 전환율을 추적해야 합니다.
